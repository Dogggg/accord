Remarks

The Support Vector Machine is by nature a binary classifier. One of the ways to extend the original SVM algorithm to multiple classes is to build a one- against-one scheme where multiple SVMs specialize to recognize each of the available classes. By using a competition scheme, the original multi-class classification problem is then reduced to n*(n/2) smaller binary problems.

Currently this class supports only Kernel machines as the underlying classifiers. If a Linear Support Vector Machine is needed, specify a Linear kernel in the constructor at the moment of creation.

Examples

CopyC#
 1// Sample data
 2//   The following is simple auto association function
 3//   where each input correspond to its own class. This
 4//   problem should be easily solved by a Linear kernel.
 5
 6// Sample input data
 7double[][] inputs =
 8{
 9    new double[] { 0 },
10    new double[] { 3 },
11    new double[] { 1 },
12    new double[] { 2 },
13};
14
15// Output for each of the inputs
16int[] outputs = { 0, 3, 1, 2 };
17
18
19// Create a new Linear kernel
20IKernel kernel = new Linear();
21
22// Create a new Multi-class Support Vector Machine with one input,
23//  using the linear kernel and for four disjoint classes.
24var machine = new MulticlassSupportVectorMachine(1, kernel, 4);
25
26// Create the Multi-class learning algorithm for the machine
27var teacher = new MulticlassSupportVectorLearning(machine, inputs, outputs);
28
29// Configure the learning algorithm to use SMO to train the
30//  underlying SVMs in each of the binary class subproblems.
31teacher.Algorithm = (svm, classInputs, classOutputs, i, j) =>
32    new SequentialMinimalOptimization(svm, classInputs, classOutputs);
33
34// Run the learning algorithm
35double error = teacher.Run();
